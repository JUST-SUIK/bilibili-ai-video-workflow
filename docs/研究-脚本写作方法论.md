# 知识分享类短视频脚本写作方法论

> 本文档为B站知识分享类视频建立完整的脚本写作方法论，涵盖结构、钩子、爆点、节奏、结尾、平台特色及案例拆解。
> 最后更新：2026年6月24日

## 一、脚本结构方法论

### 1.1 经典结构一览

#### 结构A：钩子-痛点-方案-验证（HPSV）

```
[0:00-0:15] 钩子：抛出惊人事实或反直觉问题
[0:15-1:30] 痛点：放大观众的真实困扰
[1:30-5:30] 方案：逐步讲解解决方案
[5:30-6:30] 验证：用实际案例/数据证明方案有效
[6:30-7:00] 总结+CTA
```

- 优点：逻辑清晰，观众容易跟随；痛点共鸣强，留存高
- 缺点：如果痛点不够痛，开场会显得平淡
- 适用：工具教程、效率提升类、问题解决类

#### 结构B：故事-原理-迁移（SPM）

```
[0:00-1:00] 故事：讲一个真实/虚构的有趣故事
[1:00-4:00] 原理：从故事中提炼出核心知识/原理
[4:00-6:30] 迁移：告诉观众如何将原理应用到自己身上
[6:30-7:00] 总结+CTA
```

- 优点：故事天然有吸引力，记忆点强
- 缺点：故事和知识的衔接如果生硬，会显得割裂
- 适用：概念科普、思维方式类、深度分析类

#### 结构C：对比-拆解-结论（DAC）

```
[0:00-0:30] 对比：提出两个/多个方案的对立
[0:30-5:00] 拆解：逐一对比分析各个方案
[5:00-6:30] 结论：给出明确推荐
[6:30-7:00] 总结+CTA
```

- 优点：天然有冲突感，观众有"站队"欲望，弹幕互动强
- 缺点：需要对多个方案都有深入了解
- 适用：工具对比、方法对比、观点辩论类

#### 结构D：悬念-层层剥洋葱（SOO）

```
[0:00-0:20] 悬念：抛出一个让人好奇的问题
[0:20-2:00] 第一层：表面原因
[2:00-4:00] 第二层：深层原因
[4:00-6:00] 第三层：本质原因
[6:00-7:00] 真相揭示+CTA
```

- 优点：好奇心驱动，留存率极高
- 缺点：需要内容本身有足够的"层次"可以挖掘
- 适用：揭秘类、深度分析类、"为什么"类选题

#### 结构E：Before-After-Bridge（BAB）

```
[0:00-1:00] Before：展示"没有这个知识"时的糟糕状态
[1:00-2:00] After：展示"掌握这个知识"后的理想状态
[2:00-6:00] Bridge：搭建从Before到After的桥梁（具体方法）
[6:00-7:00] 总结+CTA
```

- 优点：前后对比强烈，观众有明确的"获得感"
- 缺点：After部分如果太夸张，会显得不真实
- 适用：技能提升、工具教程、个人成长类

### 1.2 最适合5-7分钟短视频的结构

**推荐结构：改良版HPSV（钩子-痛点-方案-验证）**

理由：
1. 5-7分钟时间有限，需要快速进入正题，HPSV的钩子+痛点组合能在前30秒锁定观众
2. 方案部分是主体，可以灵活拆分为2-3个要点，每个要点1-2分钟
3. 验证部分可以用一个简洁的案例收尾，不需要展开太多
4. 整体节奏紧凑，信息密度高

**5-7分钟时间分配参考：**

```
钩子：     0:00 - 0:15  （15秒，约75字）
痛点共鸣： 0:15 - 1:00  （45秒，约225字）
核心方案： 1:00 - 5:00  （4分钟，约1200字，分3个要点）
案例验证： 5:00 - 6:15  （75秒，约375字）
总结+CTA： 6:15 - 7:00  （45秒，约225字）
```

## 二、开头钩子（Hook）技术

### 2.1 前5秒法则

B站观众的注意力窗口极短。数据显示，知识区视频的平均跳出集中在前5-10秒。前5秒必须做到以下至少一点：

- 引发好奇（"你知道吗..."）
- 制造悬念（"结果让我震惊..."）
- 承诺价值（"看完这个视频你就能..."）
- 引发共鸣（"你是不是也遇到过..."）
- 颠覆认知（"其实XX根本不是你想的那样"）

### 2.2 十种经典钩子类型

| 序号 | 类型 | 机制 | 适合场景 |
|------|------|------|----------|
| 1 | 悬念钩 | 抛出未解之谜 | 深度分析、揭秘类 |
| 2 | 反直觉钩 | 颠覆常识的认知冲击 | 科普、概念类 |
| 3 | 痛点钩 | 直击观众的真实困扰 | 教程、效率类 |
| 4 | 数据钩 | 用惊人数据开场 | 趋势分析、行业洞察 |
| 5 | 故事钩 | 用一句话开启一个故事 | 经验分享、案例类 |
| 6 | 提问钩 | 抛出观众想知道答案的问题 | 科普、对比类 |
| 7 | 场景钩 | 描述一个观众熟悉的场景 | 工具教程、生活类 |
| 8 | 对比钩 | 制造对立或冲突 | 工具对比、观点类 |
| 9 | 成果钩 | 直接展示最终成果 | 教程、实战类 |
| 10 | 热点钩 | 借势当下热点话题 | 时事评论、快速响应类 |

### 2.3 AI/科技主题钩子示例（10个）

**示例1 - 反直觉钩：**
> "我让ChatGPT写了一个程序，结果它写出的代码，连它自己都解释不了。这到底是怎么回事？"

**示例2 - 数据钩：**
> "2026年，全球已经有超过3亿人每天在用AI写代码。但你知道吗，其中超过60%的人，用的方式都是错的。"

**示例3 - 痛点钩：**
> "你有没有这种感觉：明明AI工具就在手边，但每次用出来的结果，就是不如别人发的那些效果？"

**示例4 - 悬念钩：**
> "上周我做了一个实验：让AI完全替代我工作一整天。结果，有些事它做得比我好10倍，但有一件事，它彻底搞砸了。"

**示例5 - 提问钩：**
> "一个问题：如果你只能选一个AI工具用一辈子，你选ChatGPT还是Claude？先别急着回答，看完这个视频你可能会改变想法。"

**示例6 - 对比钩：**
> "所有人都在说Prompt Engineering要被淘汰了。但我今天要告诉你，不是它要淘汰，是90%的人根本没学会怎么用它。"

**示例7 - 场景钩：**
> "凌晨两点，你还在改周报。领导发来消息说'明天早上要'。这时候你打开AI，输入一句话，10分钟后，一周的工作总结就写完了。"

**示例8 - 故事钩：**
> "三个月前，我还是一个连Python都写不利索的计算机学生。今天，我用AI搭了一个完整的网站，全程没手写一行代码。"

**示例9 - 成果钩：**
> "（直接展示一个精美的AI生成画面）这个画面，从构思到完成，只用了5分钟。今天我就把完整的方法分享给你。"

**示例10 - 热点钩：**
> "昨天OpenAI又发新模型了，朋友圈刷屏了一整天。但大多数人连旧模型都没用明白。今天咱们不追热点，把基础打扎实。"

### 2.4 钩子写作检查清单

- [ ] 前5秒是否能让观众产生"我需要知道答案"的感觉？
- [ ] 是否承诺了明确的价值（看完能获得什么）？
- [ ] 语言是否口语化，像在和朋友聊天？
- [ ] 是否避免了"大家好我是XXX"这类消耗注意力的开场？
- [ ] 是否和标题形成呼应（不要标题党）？

## 三、爆点设计

### 3.1 什么是"爆点"

爆点是视频中让观众产生强烈情绪反应、想要发弹幕、评论或转发的瞬间。它不是偶然出现的，而是可以被设计和预埋的。

### 3.2 爆点的五种常见模式

#### 模式一：反直觉爆点

机制：打破观众的既有认知，制造"原来如此"的恍然大悟感。

触发弹幕类型："卧槽"、"真的假的"、"涨知识了"、"颠覆认知"

设计方法：
- 先建立一个"常识"（大家都这么认为）
- 然后用事实/数据/逻辑推翻它
- 最后给出新的理解

示例旁白：
> "很多人以为大语言模型是在'理解'你说的话。其实不是。它只是在做一件事：预测下一个最可能的词。就像一个超级高级的自动补全。"

#### 模式二：共鸣爆点

机制：精准描述观众经历过但没说出来的感受，引发"这说的就是我"的认同。

触发弹幕类型："太真实了"、"说到心坎里了"、"我就是这样"、"被精准打击了"

设计方法：
- 观察目标受众的日常痛点
- 用具体、生动的语言描述场景
- 不要抽象，要具象到细节

示例旁白：
> "你有没有试过，精心写了一段Prompt，发给AI，结果它回了一大段正确但完全没用的废话？你又改了改，它又给了一段新的废话。来回五六次，你终于放弃了，觉得自己可能不适合用AI。"

#### 模式三：争议爆点

机制：提出一个有争议的观点，让观众忍不住想表达自己的看法。

触发弹幕类型："不同意"、"说得对"、"看情况吧"、"终于有人说了"

设计方法：
- 找到一个有两派观点的话题
- 明确表态，不要含糊
- 给出理由但留有讨论空间

示例旁白：
> "我说一个可能得罪人的观点：学AI不需要学编程。很多人不同意，但你想想，你用手机拍照需要懂光学原理吗？"

#### 模式四：实用爆点

机制：给出一个立刻能用的具体方法/工具/模板，让观众觉得"太有用了，必须收藏"。

触发弹幕类型："收藏了"、"已三连"、"求模板"、"太实用了"

设计方法：
- 提供一个可以直接复制使用的Prompt模板
- 展示一个具体的操作步骤
- 给出一个意想不到的工具推荐

示例旁白：
> "这里我给你一个万能Prompt模板，你直接复制就能用：'你是一个[角色]，请帮我[任务]，要求[标准]，输出格式是[格式]。'记住这个结构，90%的场景都能覆盖。"

#### 模式五：情感爆点

机制：通过个人真实经历或对未来的展望，触动观众的情感。

触发弹幕类型："加油"、"说得真好"、"有被激励到"、"破防了"

设计方法：
- 分享真实的学习/成长经历
- 表达对某个趋势的真实感受
- 用有画面感的语言描述

示例旁白：
> "说实话，刚开始学AI的时候我也很焦虑。感觉每天都有新工具出来，学不完。后来我想通了：你不需要学会所有工具，你只需要找到最适合你的那一个，然后把它用到极致。"

### 3.3 爆点预埋策略

在5-7分钟的视频中，建议预埋3-5个爆点，分布如下：

```
[0:00-0:15] 爆点1：钩子本身就是爆点（反直觉或共鸣）
[1:00-2:00] 爆点2：共鸣爆点（描述痛点时）
[3:00-4:00] 爆点3：反直觉或实用爆点（核心讲解中）
[5:00-6:00] 爆点4：实用爆点（给出具体方法/模板）
[6:00-7:00] 爆点5：情感爆点（结尾升华）
```

### 3.4 爆点写作检查清单

- [ ] 每个爆点是否能在10秒内引发情绪反应？
- [ ] 爆点之间是否有足够的间隔（不要堆叠）？
- [ ] 爆点是否和内容主题紧密相关（不是为了爆而爆）？
- [ ] 是否有至少一个"实用爆点"（提升收藏率）？
- [ ] 是否有至少一个"共鸣爆点"（提升评论率）？

## 四、节奏控制

### 4.1 5-7分钟视频的节奏模型

知识分享视频的节奏不是匀速的，而是有起伏的"波浪型"：

```
情绪/注意力
    ^
    |    *         *         *
    |   / \       / \       / \
    |  /   \     /   \     /   \
    | /     \   /     \   /     \
    |/       \ /       \ /       \  *
    +---------------------------------->
    0   1   2   3   4   5   6   7 (分钟)
    钩子  痛点  核心1  核心2  核心3  验证  CTA
```

每个"波峰"对应一个爆点或高能时刻，每个"波谷"是过渡或铺垫。

### 4.2 信息密度最佳配比

5-7分钟视频（约1500-2100字旁白）的信息密度建议：

| 内容类型 | 占比 | 作用 | 字数参考 |
|----------|------|------|----------|
| 钩子+痛点 | 15% | 锁定注意力 | 225-315字 |
| 核心知识点 | 50% | 传递价值 | 750-1050字 |
| 案例/演示 | 20% | 具象化理解 | 300-420字 |
| 过渡/铺垫 | 10% | 让观众喘口气 | 150-210字 |
| 总结+CTA | 5% | 收尾引导 | 75-105字 |

### 4.3 避免中途流失的七种技巧

**技巧一：每60-90秒制造一个小高潮**

不要让观众连续听超过90秒的"平铺直叙"。每隔60-90秒，需要有一个：
- 有趣的例子
- 反直觉的观点
- 实用的技巧
- 幽默的吐槽

**技巧二：使用"路标语言"**

告诉观众"我们在哪里"和"接下来去哪里"：

> "好，刚才我们搞清楚了XX是什么。接下来是最关键的部分——怎么用它。"

> "这是第一点，还有两个更重要的。"

> "最后一点，也是我认为最有价值的。"

**技巧三：信息块不超过3分钟**

任何一个知识点的连续讲解不超过3分钟。如果内容太多，拆分成多个子点，每个子点用一个具体例子支撑。

**技巧四：变换节奏**

```
快（兴奋语气，展示成果）
    → 慢（沉稳语气，解释原理）
        → 快（举例，加快语速）
            → 慢（总结，降低语速）
```

**技巧五：设置"开放式问题"**

在视频中段抛出一个问题，但不立刻回答，告诉观众"一会儿揭晓"。这能制造微小的悬念，维持好奇心。

**技巧六：视觉节奏配合**

- 信息密度高的段落：画面切换快，字幕多，动态元素多
- 需要观众思考的段落：画面简洁，节奏放缓
- 爆点时刻：画面特效、音效加强

**技巧七：适当留白**

不要把每一秒都填满。适当的停顿（0.5-1秒）能让重要信息被吸收，也能制造节奏感。

### 4.4 节奏控制检查清单

- [ ] 是否每60-90秒有一个"能量点"？
- [ ] 是否使用了路标语言引导观众？
- [ ] 信息块是否控制在3分钟以内？
- [ ] 是否有节奏变化（快慢交替）？
- [ ] 是否避免了连续超过2分钟的纯讲解？

## 五、结尾设计

### 5.1 知识分享类视频的五种结尾模式

#### 模式一：金句总结

用一句精炼、有力的话总结全片核心观点，让观众记住。

> "记住：AI不是来取代你的，是来放大你的。会用AI的人，才是未来最有竞争力的人。"

适用：观点类、深度分析类

#### 模式二：行动清单

给观众一个清晰的行动步骤，让他们看完就能执行。

> "今天的内容总结成三步：第一步，打开XX工具；第二步，输入这个Prompt模板；第三步，根据结果微调。就这么简单。"

适用：教程类、工具类

#### 模式三：开放式思考

抛出一个开放性问题，引导观众在评论区讨论。

> "最后我想问大家一个问题：你觉得AI发展到最后，是会让人类更懒，还是更聪明？评论区告诉我你的看法。"

适用：观点类、趋势分析类

#### 模式四：对比首尾呼应

回到开头的钩子/故事，形成闭环。

> "还记得开头我说的那个AI写不出自己能解释的代码吗？其实答案很简单——因为它从来就不是在'写'代码，它只是在'拼'代码。理解了这一点，你就理解了AI的本质。"

适用：故事类、揭秘类

#### 模式五：价值升华

从具体知识上升到更高的价值层面。

> "今天讲的这些技巧，说到底就是一句话：学会和AI对话，就像学会和一个新同事沟通一样。它很聪明，但它需要你告诉它要什么。这个能力，可能是我们这一代人最重要的技能之一。"

适用：所有类型，特别是想建立个人品牌的视频

### 5.2 CTA（行动号召）设计

#### CTA的层次设计

不要一次要求太多，选择1-2个主要CTA：

| 优先级 | CTA类型 | 话术示例 | 目标 |
|--------|---------|----------|------|
| 最高 | 关注 | "如果觉得有用，点个关注，我会持续分享AI干货" | 涨粉 |
| 高 | 三连 | "觉得有帮助的话，一键三连支持一下" | 互动数据 |
| 中 | 收藏 | "建议收藏，以后用到的时候翻出来" | 收藏率 |
| 中 | 评论 | "评论区告诉我你最想学哪个工具" | 评论率 |
| 低 | 转发 | "如果身边有人也需要，转发给TA" | 传播 |

#### CTA的最佳位置

- 片尾（必须有）
- 中间自然过渡处（可选，但效果好）
- 给出实用内容后立刻跟（"这个模板给你了，记得收藏"）

#### CTA的语气

不要用命令式，要用"利他"式：

- 错："点赞关注收藏！"
- 对："如果对你有帮助，点个关注，后面还有更多AI干货分享。"

### 5.3 下期预告设计

#### 预告的三种方式

**方式一：悬念式**
> "下期视频，我会演示一个让我自己都震惊的AI用法。先不剧透，关注等更新。"

**方式二：系列式**
> "这是AI入门系列的第二期。下一期，我们聊聊AI Agent——可能是今年最重要的AI概念。"

**方式三：互动式**
> "下期视频做什么？评论区投票：A. AI绘画入门 B. Prompt进阶技巧 C. AI编程实战。票数最高的我下周就做。"

### 5.4 结尾写作检查清单

- [ ] 结尾是否有明确的收束感（不要戛然而止）？
- [ ] CTA是否只选了1-2个（不要贪多）？
- [ ] CTA的语气是否自然（不要像念广告词）？
- [ ] 是否有下期预告或系列感？
- [ ] 结尾是否有记忆点（金句/升华）？

## 六、B站特色要素

### 6.1 B站观众的偏好和习惯

#### 用户画像

- 年龄：18-35岁为主，Z世代占比高
- 学历：大学生和刚工作的人群占大比例
- 特点：反感硬广，喜欢真诚分享；追求"有料"而非"高大上"
- 消费习惯：愿意花时间看有深度的内容，但容忍度有上限

#### 内容偏好

1. **真实感 > 包装感**：B站观众对过度包装的内容有天然抵触。用真实的学习经历、踩坑经验比"我教你成功"更受欢迎
2. **有料 > 有趣**：有趣是加分项，但有料是基础。纯搞笑的知识视频在B站活不长
3. **深度 > 广度**：一个知识点讲透比十个知识点蜻蜓点水更好
4. **平等感 > 权威感**：B站观众喜欢"和我一样的人在分享"，不喜欢"专家在教训我"

#### 观看习惯

- 高峰时段：午休（12:00-13:30）、晚间（20:00-23:00）、周末全天
- 平均观看时长：知识区优质视频完播率在30-50%（5-7分钟视频）
- 收藏习惯：B站用户的收藏率远高于其他平台，实用内容容易被收藏
- 倍速观看：约40%的用户会使用1.25x-1.5x倍速

### 6.2 弹幕互动设计

#### 容易引发弹幕的内容类型

| 类型 | 示例 | 预期弹幕 |
|------|------|----------|
| 反直觉观点 | "其实XX不是你想的那样" | "？？？"、"真的假的" |
| 精准吐槽 | "是不是你的日常？" | "太真实了"、"被监控了" |
| 争议表态 | "我觉得A比B好" | "不同意"、"确实" |
| 求助互动 | "你们觉得呢？" | 各种回答 |
| 预言验证 | "我猜你们会说..." | "你怎么知道"、"确实" |
| 共鸣场景 | 描述一个共同经历 | "破防了"、"说的是我" |
| 实用干货 | "这个方法直接拿去用" | "收藏了"、"学到了" |

#### 弹幕引导技巧

**技巧一：主动预测弹幕**
> "我知道你们现在在想什么——'这不就是XXX吗？'还真不是，听我说完。"

**技巧二：设置投票点**
> "觉得AI会取代程序员的扣1，不会的扣2。"

**技巧三：留出"吐槽空间"**
故意说一个有点"离谱"的观点，给观众吐槽的机会，增加互动。

**技巧四：分P互动**
如果视频较长，可以在中间设置"下一段更精彩，别走开"的提示，减少跳出。

### 6.3 B站知识区爆款视频的共同特征

#### 数据特征

- 标题：15-25字，包含核心关键词和价值承诺
- 封面：信息量大但不杂乱，通常有大字标题+关键图示
- 时长：5-15分钟是知识区的"黄金时长"
- 发布时间：工作日晚8-10点，周末上午10-12点

#### 内容特征

1. **强开头**：前10秒必须抓住注意力，没有寒暄
2. **有节奏**：信息密度有起伏，不是匀速灌输
3. **有案例**：纯理论讲解很难火，必须有具体案例
4. **有态度**：有明确的观点和立场，不含糊
5. **有实用价值**：看完之后观众能带走点什么
6. **有记忆点**：至少一个让观众记住的金句或画面

#### 标题特征

爆款标题的常见模式：

| 模式 | 示例 |
|------|------|
| 数字+价值 | "5分钟搞懂大语言模型" |
| 问题+悬念 | "AI到底能不能取代程序员？答案出乎意料" |
| 对比+冲突 | "ChatGPT vs Claude：到底谁更强？" |
| 痛点+方案 | "还在手动写周报？AI 10分钟搞定" |
| 身份+经历 | "计算机学生学AI三个月，分享我的真实感受" |
| 反常识+揭秘 | "为什么你用AI总是得不到好结果？99%的人不知道这一点" |

#### 封面设计特征

- 大字标题（手机端能看清）
- 对比色强烈（吸引眼球）
- 有关键图示或人物表情
- 信息量适中（不要太满也不要太空）
- 风格统一（形成系列感）

## 七、脚本拆解案例

### 案例一：科普讲解类

**视频类型**：概念科普
**时长**：约6分钟
**结构**：HPSV（钩子-痛点-方案-验证）

**脚本结构拆解：**

```
[0:00-0:12] 钩子（反直觉钩）
"你以为AI在'思考'？其实它只是在做一件事——猜下一个字。"

[0:12-0:45] 痛点共鸣
"很多人用AI用不好，根本原因是不理解AI的工作方式。
你以为它像人一样思考，所以给它的指令也像跟人说话一样。
但AI不是人。它有自己的'思维方式'。"

[0:45-2:30] 核心讲解1：原理通俗化
- 用"超级自动补全"的比喻解释大语言模型
- 用"猜词游戏"让观众直观感受

[2:30-4:00] 核心讲解2：实际影响
- 解释为什么AI会"一本正经地胡说八道"
- 解释为什么Prompt的写法这么重要
- 给出2-3个具体的例子

[4:00-5:20] 核心讲解3：实用建议
- 理解了原理之后，使用AI的3个关键原则
- 每个原则配一个具体例子

[5:20-6:00] 总结+CTA
"记住：AI不是人，不要用跟人说话的方式跟它说话。
理解了这一点，你就超过了90%的AI用户。
觉得有用就关注，下期讲怎么写出让AI听话的Prompt。"
```

**为什么能火：**
1. 钩子直接颠覆认知，前5秒锁定注意力
2. 用"超级自动补全"的比喻，把复杂概念简单化
3. 痛点精准——"用AI用不好"是普遍困扰
4. 有实用价值——看完立刻能改善使用方式
5. 结尾有系列感，引导关注

### 案例二：工具对比类

**视频类型**：工具评测对比
**时长**：约8分钟
**结构**：DAC（对比-拆解-结论）

**脚本结构拆解：**

```
[0:00-0:10] 钩子（对比钩）
"ChatGPT和Claude，到底该用哪个？今天我用10个真实场景测给你看。"

[0:10-0:40] 建立对比框架
"我从三个维度来对比：写代码、写文案、逻辑推理。
每个维度我都会用同一个任务测试，结果可能会让你意外。"

[0:40-2:30] 对比维度1：写代码
- 同一个编程任务分别给两个工具
- 展示代码质量、解释清晰度
- 【爆点】：某一方在某个细节上明显更好

[2:30-4:20] 对比维度2：写文案
- 同一个文案需求分别测试
- 展示语言风格、创意程度
- 【爆点】：两者的风格差异让人意外

[4:20-6:00] 对比维度3：逻辑推理
- 同一个逻辑题分别测试
- 展示推理过程、答案准确度
- 【爆点】：在某个反直觉的题目上结果出人意料

[6:00-7:20] 综合评分+使用建议
- 三个维度的综合评分表
- 不同场景的推荐选择
- 【实用爆点】：给出一个决策流程图

[7:20-8:00] 总结+CTA
"没有最好的工具，只有最适合的场景。
我的建议是：两个都试试，找到你自己的偏好。
收藏这个视频，下次纠结的时候翻出来看。"
```

**为什么能火：**
1. 对比天然有冲突感，观众有"站队"欲望
2. "10个真实场景"的具体承诺，让观众觉得内容扎实
3. 每个对比维度都有爆点，节奏紧凑
4. 最后给出明确建议，实用性强
5. 决策流程图是"可截图保存"的内容，收藏率高

### 案例三：个人经验分享类

**视频类型**：学习经验/成长分享
**时长**：约7分钟
**结构**：SPM（故事-原理-迁移）

**脚本结构拆解：**

```
[0:00-0:15] 钩子（故事钩）
"三个月前我做了一个决定：每天用AI学习2小时。
三个月后的今天，我来告诉你这个实验的结果。"

[0:15-1:30] 故事铺垫
- 实验的起因（焦虑、迷茫、不知道学什么）
- 实验的设计（每天做什么、怎么记录）
- 【共鸣爆点】：描述焦虑的感受

[1:30-3:00] 第一个月：迷茫期
- 踩了哪些坑
- 产生了哪些误区
- 【反直觉爆点】：某个认知被颠覆

[3:00-4:30] 第二个月：开窍期
- 找到了适合自己的方法
- 关键转折点是什么
- 【实用爆点】：分享具体的学习方法

[4:30-5:30] 第三个月：收获期
- 具体的成果（能做什么了）
- 最大的感受
- 【情感爆点】：对学习态度的变化

[5:30-6:30] 提炼原理
- 从三个月的经历中总结出3条原则
- 这些原则适用于任何技能学习

[6:30-7:00] 总结+CTA
"学习AI最大的门槛不是技术，是心态。
不要想着一步到位，先从每天30分钟开始。
关注我，一起在AI这条路上慢慢走。"
```

**为什么能火：**
1. "三个月实验"的设定天然有故事线，观众想知道结果
2. 真实的学习经历比"教你成功"更有说服力
3. 按时间线展开，逻辑清晰
4. 有踩坑也有收获，真实可信
5. 结尾的"慢慢走"调性真诚，不贩卖焦虑

## 八、脚本写作实操清单

### 写脚本前的准备

- [ ] 明确视频的核心价值（观众看完能带走什么）
- [ ] 确定目标受众（是谁？他们的痛点是什么？）
- [ ] 选择脚本结构（HPSV/SPM/DAC/SOO/BAB）
- [ ] 列出3-5个想传递的关键点
- [ ] 准备1-2个具体案例

### 写脚本时的要点

- [ ] 先写钩子，再写正文，最后写结尾
- [ ] 每个关键点配一个具体例子
- [ ] 预埋3-5个爆点
- [ ] 使用路标语言引导观众
- [ ] 语言口语化，像在和朋友聊天
- [ ] 每句话不超过30字

### 写完后的检查

- [ ] 朗读一遍，检查是否顺口
- [ ] 计算时长（按每秒5字计算）
- [ ] 检查信息密度是否合理
- [ ] 检查爆点分布是否均匀
- [ ] 检查结尾是否有记忆点和CTA
- [ ] 请别人听一遍，看能否跟上逻辑

### 脚本字数与时长对照表

| 时长 | 旁白字数（含停顿） | 有效信息字数 |
|------|---------------------|--------------|
| 5分钟 | 1500字 | 1200字 |
| 6分钟 | 1800字 | 1440字 |
| 7分钟 | 2100字 | 1680字 |
| 8分钟 | 2400字 | 1920字 |
| 10分钟 | 3000字 | 2400字 |

> 注：按正常语速每秒5字计算，有效信息字数按80%折算（去除停顿、过渡等）。
