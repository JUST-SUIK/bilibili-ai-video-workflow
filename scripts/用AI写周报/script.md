## 第二部分 — 核心方案（1:15-5:00，680字）

### 要点1：AI写周报的正确姿势（1:15-2:30，220字）

#### 旁白
好，痛点说完了，我们来说解决方案。

但我要先告诉你一个很多人不知道的事：AI写周报，不是让它"帮你写"。

是让它"帮你润色"。

什么意思？

你不能直接跟AI说："帮我写本周周报。"

它不知道你这周干了啥。

你得先告诉它。

所以正确的流程是：

第一步，你自己花五分钟，快速回忆这周做了什么。

用关键词就行，不用写完整句子。

比如：周一开会、周三交付方案、帮同事解决问题。

这五分钟，省不了。

因为只有你知道这周发生了什么。

**运动方式：** 中等节奏，重点在流程对比

## 第三部分 — 验证对比（5:00-6:00，180字）

光说不练假把式，我们来看Before和After。

**Before：自己写**

"周一开会，周三交方案，周四帮同事解决问题。"

流水账，领导看了想睡觉。

**After：AI润色后**

"本周重点推进Q3目标对齐，完成产品方案V2交付，顺手帮同事解决了服务器问题。"

同样的内容，但听起来专业多了。

区别在哪？

你负责"做了什么"，AI负责"怎么说"。

你不用再纠结"改了几个bug"还是"修复漏洞"。

你把事实告诉AI，它帮你包装。

这就是AI写周报的正确用法。

## Prompt模板附录（视频中展示）

### 模板1：基础版
```
请根据以下关键词，帮我整理成一份工作周报：
【工作关键词】
- 周一：[具体事项]
- 周二：[具体事项]
- 周三：[具体事项]
- 周四：[具体事项]
- 周五：[具体事项]

【输出要求】
1. 工作内容分点列出
2. 突出关键成果
3. 包含下周计划
4. 语气专业但不生硬
```

### 模板2：项目版
```
请根据以下项目进展，帮我写一份项目周报：
【项目名称】[项目名称]
【本周进展】
- 完成了：[具体任务]
- 遇到的问题：[问题描述]
- 解决方案：[解决方式]
【下周计划】
- [计划事项]

【输出要求】
1. 重点突出项目进度
2. 问题和解决方案要对应
3. 下周计划要具体可执行
```

### 模板3：问题解决版
```
请根据以下问题解决过程，帮我写一份技术周报：
【问题描述】[问题是什么]
【解决步骤】
1. [步骤1]
2. [步骤2]
3. [步骤3]
【最终结果】[结果如何]
【经验总结】[学到了什么]

【输出要求】
1. 问题描述要清晰
2. 解决步骤要详细
3. 突出解决能力和技术经验
```
