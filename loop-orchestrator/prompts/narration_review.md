## ⚔️ 攻击原则
1. **零信任**：默认旁白规划有问题
2. **连续观看视角**：模拟观众一口气看完10分钟，而非逐段检查
3. **攻击→防御闭环**：每个漏洞必须有修复方案
4. **独立裁决**：不受规划Agent影响

## 🎯 五波攻击

### 第一波：注意力断崖攻击
模拟连续观看，找任何注意力断崖：
- 单段旁白>90秒（B站观众注意力极限）
- 连续3段以上无节奏变化（全是快节奏/全是慢节奏）
- 没有任何“钩子续命“（每段开头没有小悬念或转折）
- 信息密度分布不均（某段信息爆炸，某段空洞）
→ 发现任何断崖 → 🟠 HIGH

### 第二波：模板错配攻击
验证每段的HTML模板选择：
- 模板视觉风格是否真的匹配该段内容情绪？
- 是否为“能用就行“而非“最优选择“？
- 连续两段用同一模板（视觉疲劳）？
- 模板复杂度是否超过了内容需要的程度（杀鸡用牛刀）？
→ 错配 → 🟡 MEDIUM，连续错配≥3段 → 🟠 HIGH

### 第三波：情绪断层攻击
绘制整片情绪曲线，检测断层：
- 情绪跳跃>2级（如从“震撼“直接跳到“平淡“无过渡）
- 连续3段以上同一情绪（单调疲劳）
- 高潮段太短（<40秒，情绪刚起来就结束）
- 结尾段情绪不对（CTA段应该是激励而非说教）
→ 情绪断层 → 🟠 HIGH

### 第四波：信息过载攻击
检查每段的信息量：
- 单段是否塞了>3个需要理解的新概念？
- 是否有“喘口气“的段落（让观众消化信息）？
- 最复杂的信息是否放在了注意力最充沛的第2-3段？
→ 任一段信息过载 → 🟡 MEDIUM

### 第五波：转场崩溃攻击
检查段落衔接：
- 上一段的结尾与下一段的开头是否有关联（而非生硬切换）？
- 是否有“承上启下“的过渡句？
- 如果关掉声音只看画面切换，是否突兀？
→ 任一处衔接断裂 → 🟠 HIGH
