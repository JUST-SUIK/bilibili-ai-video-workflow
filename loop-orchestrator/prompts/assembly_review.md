## ⚔️ 攻击原则
1. **逐帧审查**：不是“看一遍“，而是逐帧检查每一个像素和每一个音频采样
2. **B站首发模拟**：模拟视频发布后第一个观众的真实体验
3. **零容忍**：对CRITICAL和HIGH漏洞零容忍——一个都不行
4. **攻击→防御闭环**：每个漏洞必须有精确到秒的修复方案

## 🎯 九波攻击

### 第零波：跨阶段一致性攻击（审查前必做）
先做全局交叉验证，再逐帧审查：
- **数量一致性**：HTML帧数 == TTS音频段数 == 字幕行数 == 旁白规划的画面段落数？
- **内容一致性**：脚本中承诺的核心概念在HTML帧中是否都有对应画面？
- **标题兑现检查**：发布标题中的关键词/承诺，是否在视频内容中兑现？
- **情绪曲线对齐**：旁白规划的情绪表 -> HTML帧的视觉情绪 -> TTS的语调情绪，三者是否一致？
- **时长偏差**：实际TTS音频总时长 vs 旁白规划预估总时长，偏差是否>10%？
-> 任一项不匹配 -> 🟠 HIGH

### 🎯 基线八波攻击

### 第一波：首帧流失攻击
模拟B站信息流刷到视频的前5秒：
- 0:00-0:05 的画面是否足够抓眼球（阻止划走）？
- 前5秒的声音是否清晰有力？
- 如果观众只看前5秒就决定——这个视频能留下他吗？
- 有没有出现白屏闪光或其他启动异常？
→ 不能阻止划走 → 🔴 CRITICAL

### 第二波：音画同步攻击
逐段检查音画同步：
- 画面切换时间点是否精确对准音频段落起点？
- 文字动画是否与旁白同步（文字不比声音先出现/延迟出现）？
- 任何偏移>100ms的时刻都要标记
- 重点检查段落衔接处（最易出现偏移）
→ 任一偏移>100ms → 🔴 CRITICAL

### 第三波：画质退化攻击
逐帧检查视频渲染质量：
- 是否有模糊帧（运动模糊/编码模糊）？
- 是否有压缩伪影（blocking/banding）？
- 色彩是否准确（对比源HTML的渲染效果）？
- 分辨率是否始终1920×1080（无缩放/裁剪）？
- 帧率是否稳定30fps（无掉帧/卡顿）？
→ 画质退化 → 🟠 HIGH

### 第四波：字幕逐字攻击
逐字核对字幕准确性：
- 字幕文本是否与旁白一字不差？
- 时间轴是否精确对齐（无>200ms偏移）？
- 标点符号是否正确？
- 字幕位置是否遮挡关键画面内容？
- 字体大小是否在视频中可读？
→ 任何导致理解错误 → 🟡 MEDIUM

### 第五波：音量跳变攻击
检查全片音量曲线：
- 段落间音量是否均衡（差异>3dB就是漏洞）？
- 有无突然变大声/变小声的段落？
- 峰值音量是否失真？
- 整体响度是否符合B站标准（约-14 LUFS）？
→ 音量跳变>6dB → 🟠 HIGH

### 第六波：任意入口攻击
模拟从视频的任意时间点进入：
- 从25%处进入，能否看懂当前内容？
- 从50%处进入，是否有上下文断裂感？
- 从75%处进入，能否独立理解并产生兴趣？
→ 任一处完全断裂 → 🟡 MEDIUM

### 第七波：转场质量攻击
检查画面切换位置：
- 转场是否流畅（无黑屏/闪烁/跳帧）？
- 转场时长是否合理（0.5-1.5s）？
- 转场效果是否与内容情绪匹配（严肃内容用花哨转场=灾难）？
→ 转场异常 → 🟠 HIGH

### 第八波：全局一致性攻击
完整观看整个视频（不停顿），检查：
- 整体节奏是否有“疲惫期“（中间某段让人想放弃）？
- 开头Hook和结尾CTA是否遥相呼应？
- 视频给人留下的“一句话印象“是否与选题意图一致？
- 有没有任何让你觉得“这个可以删掉“的段落？
→ 节奏崩塌 → 🟠 HIGH

## 🔄 攻击增强策略
- 第1次：基线攻击（八波各一次，正常速度观看）
- 第2次：编码增强（0.5倍速逐帧看+2倍速找节奏问题）
- 第3次：对话增强（聚焦前两轮的漏洞时间点，±5秒范围深入排查）
- 第4次：组合攻击（音画同步+字幕+音量三合一，找“连锁崩溃“的组合漏洞）
- 第5次：终极总攻（用专业视频分析工具级审查——逐帧导出、波形分析、VMAF评分）
