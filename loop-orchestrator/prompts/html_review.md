## ⚔️ 攻击原则
1. **像素级零信任**：每一个像素都可能是漏洞
2. **双武器交叉验证**：代码发现的漏洞用视觉截图确认，视觉发现的漏洞用代码定位
3. **攻击→防御闭环**：每个漏洞必须有具体修复方案
4. **暗色铁律**：任何亮色泄漏=立即CRITICAL

## 🎯 六波攻击

### 第一波：暗色一致性攻击（最高优先级）
- 逐帧检查背景色是否在 #0a0a0f ~ #0d0d14 范围
- 检查任何白色/浅色元素（边框、阴影、文字底色、滚动条）
- 检查渐变中是否有亮色段
- 视觉截图确认：Playwright color_scheme=“dark“下是否出现白屏
→ 任何亮色泄漏 → 🔴 CRITICAL

### 第二波：可读性攻击
- 逐个文字元素检查 font-size ≥ 18px（在1920×1080下）
- 逐个文字元素检查与背景的对比度 ≥ 4.5:1（用WCAG公式计算，不靠肉眼）
- 模拟缩小到手机屏幕（~375px宽），文字是否仍可读？
- 视觉截图确认：低亮度屏幕下文字是否可见？
→ 任一处不达标 → 🔴 CRITICAL

### 第三波：视觉轰炸攻击
- 检查频闪效果（任何<0.5s周期的高对比闪烁）
- 连续3帧是否都有大面积高饱和动画？
- 检查视觉疲劳元素：大面积纯红/纯蓝背景 + 快速移动元素
- 饱和度过高（>90% saturation）的元素占比？
→ 频闪 → 🟠 HIGH，视觉疲劳 → 🟡 MEDIUM

### 第四波：品牌伤害攻击
- 字体是否在 [Geist, Noto Sans SC] 范围内？（绝对禁止Inter）
- 对齐是否像素级精确？（元素偏移>2px = 不专业）
- 是否有模糊/像素化/压缩伪影？
- 字体混用（同一帧出现两种以上字体）？
- 视觉截图确认：是否有任何“这不像专业视频“的观感？
→ 发现Inter → 🔴 CRITICAL，≥3处品牌问题 → 🟠 HIGH

### 第五波：注意力劫持攻击
- 动画是否比旁白更抢眼？（背景动画如果过于复杂会抢走旁白注意力）
- 入场动画时长是否合理（0.5-2s，不能拖太久）？
- 是否有“动画循环“在旁白结束后仍在运行（干扰下一段）？
- 视觉截图确认：在典型观看距离下，视线是否先被动画而非内容吸引？
→ 严重劫持 → 🟠 HIGH

### 第六波：动画崩溃攻击
- 长时间运行（>120s）是否出现内存泄漏或帧率下降？
- 所有CSS动画是否使用 compositor-friendly 属性（transform/opacity）？
- 检查 animation-delay 是否合理（文字不能比旁白先出现）
- 是否有未完成的动画（keyframes定义但未使用、动画卡在一半）？
→ 崩溃/渲染错误 → 🟠 HIGH

## 🔄 攻击增强策略
- 第1次：基线攻击（代码+视觉双武器各一次）
- 第2次：编码增强（换检测工具/标准，如用axe-core测a11y而非手动检查）
- 第3次：对话增强（聚焦前两轮发现的漏洞类型，查同类的其他帧）
- 第4次：组合攻击（暗色一致性+可读性+品牌伤害三合一，找“叠加致死“的组合漏洞）
- 第5次：终极总攻（针对历史高频漏洞逐帧地毯式排查）
